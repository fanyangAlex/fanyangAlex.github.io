2019-05-14 编程语言

词法环境

词法环境（Lexical Environments）是一种规范类型，用于根据 ECMAScript 代码的词法嵌套结构来定义标识符与特定变量和函数的关联。词法环境由一个环境记录（Environment Record）和一个可能为空的外部词法环境（outer Lexical Environment）引用组成。通常，词法环境与 ECMAScript 代码的特定语法结构相关联，例如函数声明，块声明或 Try/Catch 的 Catch 子句，并且每次执行这样的代码时都会创建新的词法环境。
一份环境记录，保存了其关联的词法环境内部的作用域中创建的标识符绑定。这份记录就成为该词法环境的环境记录。
外部环境的引用再用于对词法环境值的逻辑嵌套进行建模。词法环境的外部引用是对逻辑上围绕着内部词法环境的引用。当然，外部词法环境可能有自己的外部词法环境。词法环境可以作为多种内部词法环境的外部环境。例如，如果一个函数声明包含两个嵌套的函数声明，那么每个嵌套函数的词法环境都将以当前函数的词法环境作为其外部词法环境。
全局环境是没有外部环境的词法环境。全局环境的外部环境引用为空。全局环境的 Environment Record 可以预先填充标识符绑定，并包含关联的全局对象，其属性提供一些全局环境的标识符绑定。在执行 ECMAScript 代码时，可以向全局对象添加其他属性，并且可以修改初始属性。
模块环境是一个词法环境，它包含模块顶层声明的绑定。它还包含模块显式导入的绑定。模块环境的外部环境是一个全局环境
函数环境是对应于 ECMAScript 函数对象调用的词法环境。函数环境可能会建立新的 this 绑定。函数环境还捕获支持 super 方法调用所需的状态。

环境记录

此规范中使用了两种主要的环境记录：声明式环境记录和对象环境记录。声明式环境记录用于定义 ECMAScript 语言语法元素的效果，例如函数声明、变量声明和 CATCH 子句，这些子句直接将标识符绑定与 ECMAScript 语言值关联。对象环境记录用于定义 ECMAScript 元素(如 With 语句)的效果，该元素将标识符绑定与某些对象的属性相关联。全局环境记录和函数环境记录是专门用于脚本全局声明和函数内部顶级声明的专门化环境记录。
出于规范目的，环境记录值是记录规范类型的值，可以认为它存在于一个简单的面向对象层次结构中，其中环境记录是一个抽象类，包含三个具体的子类：声明式环境记录、对象环境记录和全局环境记录。函数环境记录和模块环境记录是声明式环境记录的子类。抽象类包括表 14 中定义的抽象规范方法。这些抽象方法对于每个具体子类都有不同的具体算法。

执行上下文

执行上下文是一种规范设备，用于跟踪 ECMAScript 实现对代码的运行时评估。在任何时候，每个代理最多都有一个实际执行代码的执行上下文。这称为代理的运行执行上下文。本规范中对正在运行的执行上下文的所有引用都表示周围代理的运行执行上下文。
执行上下文堆栈用于跟踪执行上下文。运行的执行上下文始终是此堆栈的顶部元素。每当控件从与当前运行的执行上下文关联的可执行代码转移到与该执行上下文无关的可执行代码时，都会创建新的执行上下文。新创建的执行上下文被推送到堆栈上，并成为正在运行的执行上下文。
执行上下文包含跟踪关联代码的执行进度所需的任何实现特定状态。每个执行上下文至少有表 21 中列出的状态组件。
组件|用途
–|:–:
代码执行状态|执行、挂起和恢复与此执行上下文关联的代码的任何状态。
函数|如果这个执行上下文正在计算一个函数对象的代码，那么这个组件的值就是那个函数对象。如果上下文正在计算脚本或模块的代码，则该值为 NULL。
领域|关联代码访问 ECMAScript 资源的领域记录。
脚本或模块|关联代码来源的模块记录或脚本记录。如果没有原始脚本或模块(如 InitializeHostDefinedRealm 中创建的原始执行上下文的情况)，则该值为 NULL。

由正在运行的执行上下文对代码的执行可以在本规范中定义的不同点上挂起。一旦正在运行的执行上下文被挂起，不同的执行上下文可能成为正在运行的执行上下文，并开始执行其代码。稍后，挂起的执行上下文可能再次成为正在运行的执行上下文，并在以前挂起的地方继续执行其代码。运行中的执行上下文状态在执行上下文之间的转换通常以堆栈式的后入/先出方式进行。但是，一些 ECMAScript 特性需要运行执行上下文的非 LIFO 转换。
正在运行的执行上下文的领域组件的值也称为当前领域记录。运行执行上下文的函数组件的值也称为活动函数对象。

ES2018标准翻译1-介绍

2019-05-14 编程语言

本文翻译自http://www.ecma-international.org/ecma-262/9.0/index.html

本标准定义了 ECMAScript 2018 语言,是 ECMAScript 语言的第十个版本。从 1997 年第一个版本至今，ECMAScript 已经成为全世界使用最广泛的通用编程语言，最广为人知的用途是作为浏览器内嵌语言，但也活跃在服务器和嵌入式程序中。
ECMAScript 源于几项原有的技术，最著名的包括 Javascript(Netscape)和 JScript(Microsoft)。Brendan Eich 在 Netscape 发明了该语言，并于 Navigator2.0 浏览器中第一次出现，从此之后所有浏览器，包括 Internet Explorer3.0，都包含了这项技术。
ECMAScript 语言规范的开发从 1996 年 11 月开始，第一个版本于 1997 年 7 月被 Ecma 大会收录。
该标准在 1998 年 4 月成为 ISO/IEC 16262 国际标准，为了与国际标准对齐，1998 年 7 月的 Ecma 大会批准了第二个版本即 ECMA-262。
第三个版本包含了强大的正则表达式，更好的字符串处理，新的流程控制语句，try/catch 异常处理，更严密的出错定义，数字输出格式化以及一些期待未来语言发展的次要变更。第三版于 1999 年 11 月发布，并于 2002 年发布于 ISO/IEC 16262:2002 标准。
第三版发布之后，随着 WWW 的发展，ECMAScript 得到了广泛的应用，成为了所有浏览器的默认语言。尽管为了第四版付出了巨大的工作，但该版本并没有得到发布，之后的第六版继承了其部分的内容。
第五版对浏览器实现中常见的语言规范进行了事实上的解释，并增加了对自第三版以来的新特性的支持。这些特性包括存取属性，反射创建，对象监测，属性的程序控制，附加的数组处理函数，JSON 对象的编码和解码，能够加强错误检查和程序安全的严格模式。第五版于 2009 年发布。
2011 年发布了 ECMAScript5.1。
从 2009 年开始，随着第五版的发布，第六版进入全力开发状态。在此之前，进行了大量的实验和语言增强设计工作，这些工作可以追溯到 1999 年发布的第三版。实际上，第六版的工作是十五年的积累成果。这一版的目标是为大型应用程序提供更好的支持，库创建和作为其它语言的编译目标。重大的改进包括模块，class 声明，词法块级作用域，迭代器和生成器，针对异步编程的 Promise,解构模式，尾调用优化。ECMAScript 的内置库扩展了新的数据抽象，包括 maps,sets,二进制数组，以及对字符串和正则表达式中 Unicode 字符的额外支持。通过继承内置类，可以实现更好的扩展性。第六版为定期增量的语言和库的增强打下了基础。2015 年 6 月大会正式发布了第六版。
ECMAScript 2016 是遵循 ECMA TC 39 新的年度发行版开发过程下发布的第一个 ECMAScript 版本。从 ECMAScript 2015 源代码文档构建了一个纯文本源文档，作为完全基于 GitHub 的进一步开发的基础。在这个标准开发的一年中，数以百计的请求和问题被提交，代表了数以千计的错误修复、编辑修正和其他改进。此外，还开发了许多软件工具来帮助这项工作，包括 ECMarkup、ECMarkdown 和 Gramarkdown。ES 2016 还支持一个新的指数运算符，并为 Array.Prototype 添加了一个名为 include 的新方法。
ECMAScript 2017 引入了 async 异步函数、共享内存和 Atomics 以及更小的语言和库增强、bug 修复和编辑更新。aync 异步函数通过为返回 Promise 的函数提供语法糖来改进异步编程体验。共享内存和原子引入了一种新的内存模型，允许多代理程序使用原子操作进行通信，即使在并行 CPU 上，原子操作也能确保良好的执行顺序。该规范还包括对象上的新静态方法：Object.value、Object.entry 和 Object.getOwnPropertyDescriptors。
本规范，第 9 版，通过 AsyncIterator 协议和异步生成器引入了对异步迭代的支持。该规范还包括四个新的正则表达式特性：dotAll 标志、命名捕获组、Unicode 属性转义和前瞻性断言。它还包括 REST 参数和对象属性的扩展操作符支持。也有许多小的更新，编辑和规范，了不起的社区作出了许多贡献。
许多组织和个人在 ECMA TC 39 中为开发本版本和以前的版本做出了非常重要的贡献。此外，一个充满活力的社区已经出现，为支持 TC 39 的 ECMAScript 做出了很多努力。这个社区已经审查了许多草案，归档了数千个 bug 报告，进行了实现实验，贡献了测试套件，并对全世界的开发人员社区进行了 ECMAScript 方面的教育。在这里无法列举这里做出贡献的所有人和组织。

JavaScript中的作用域

2019-05-14 编程语言

词法作用域

JavaScript 使用的是词法作用域（lexical scope），即函数执行依赖于变量作用域，该作用域是在函数定义时静态确定的，而不是调用时动态决定。为了实现这种词法作用域，Javascript 函数对象内部包含了当前作用域链的引用。

变量作用域

作用域（scope)是程序中定义变量的区域。

  • 全局作用域：全局变量，在代码的任何地方都可以访问。
  • 函数作用域：包括函数的参数，函数内部定义的变量。
  • 块级作用域：大括号封闭起来的区域，在 es6 中新增，使用 let,const 定义的变量。

规则

  • 局部变量在函数内优先级高于全局变量。
  • 声明局部变量必须使用 var，否则会声明一个全局变量。
  • 函数定义可以嵌套，因此会出现作用域嵌套。
  • let 和 const 定义的全局变量不再是全局对象的属性，而是位于名为 Script 的作用域中。

函数作用域

变量在声明他的函数体以及这个函数体嵌套的任何函数体内都是有定义的。在函数内声明的所有变量都是始终可见的，这给 JavaScript 带了一个“声明提前”的特性，即 JavaScript 函数里声明的所有变量都被提前到函数体的顶部。

作用域链

每段 JavaScript 代码都有一个与之相关联的作用域链，这个作用域链是一个对象列表或链表，这组对象定义了这段代码的作用域中的变量。
在最顶层代码中，作用域链包含一个全局对象，不嵌套的函数体内，作用域链包含全局对象和定义了函数参数和局部变量的对象，嵌套的函数体内，作用域链至少有三个对象。
当一个函数创建后，它的作用域链会被创建此函数的作用域中可访问的数据对象填充。执行此函数时会创建一个称为“运行期上下文(execution context)”的内部对象，运行期上下文定义了函数执行时的环境。每个运行期上下文都有自己的作用域链，用于标识符解析，当运行期上下文被创建时，而它的作用域链初始化为当前运行函数的[[Scope]]所包含的对象。这些值按照它们出现在函数中的顺序被复制到运行期上下文的作用域链中。它们共同组成了一个新的对象，叫“活动对象(activation object)”，该对象包含了函数的所有局部变量、命名参数、参数集合以及 this，然后此对象会被推入作用域链的前端，当运行期上下文被销毁，活动对象也随之销毁。
